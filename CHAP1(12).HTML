Database Concepts and SQL

Data

Data refers to raw, unorganized facts that need to be processed to become meaningful. Example: numbers, names, or dates.

Information

Information is processed data that is meaningful and useful for decision-making.

Features of Information

  • Accuracy
  • Relevance
  • Timeliness
  • Completeness
  • Consistency

Database and Its Purpose

A database is a structured collection of data. Its main purpose is efficient data storage, retrieval, and management.

Database Terminologies

  • Table: Collection of data in rows and columns.
  • Field: A column in a table.
  • Record: A row in a table.
  • Tuple: Another term for record.
  • Object: Data + operations in object-oriented databases.
  • Keys: Unique identifiers (Primary, Foreign, Candidate, etc.).

Data Dictionary

A data dictionary stores metadata about the structure of the database (tables, fields, data types, etc.).

Database Management System (DBMS)

Introduction

A DBMS is software for creating and managing databases.

Objectives

  • Efficient data management
  • Data integrity
  • Security and multi-user access

Advantages

  • Reduced redundancy
  • Improved data consistency
  • Easy data sharing

Disadvantages

  • High cost
  • Complex to manage
  • Performance issues for large data sets

Types of Database Models

  • Hierarchical: Tree-structured model.
  • Network: Many-to-many relationships.
  • Relational: Tables with rows and columns.
  • Entity-Relationship: Conceptual representation of data.

Integrity Constraints

  • Domain: Valid values for a field.
  • Entity: Uniqueness of rows (Primary key).
  • Referential: Foreign key references a valid row.
  • Keys: Primary, Foreign, Candidate, Composite keys.

Normalization

Introduction

Normalization organizes data to reduce redundancy and improve integrity.

Normal Forms

  • 1NF: Atomic values only, no repeating groups.
  • 2NF: Remove partial dependencies.
  • 3NF: Remove transitive dependencies.

Advantages

  • Removes redundancy
  • Improves data integrity

Disadvantages

  • More complex queries
  • Slower performance with joins

Centralized vs. Distributed Database

Introduction

Centralized stores all data at one site, distributed spreads data across locations.

Advantages

  • Centralized: Easier to manage, consistent data.
  • Distributed: Faster access, fault-tolerant.

Disadvantages

  • Centralized: Single point of failure.
  • Distributed: Complex management, data sync issues.

Comparison

Distributed databases are better for global operations, centralized for small systems.

Database Security

Introduction

Database security ensures data is protected from unauthorized access or alteration.

Challenges

  • Hackers
  • Insider threats
  • Data loss

Security Measures

  • Authentication and Authorization
  • Encryption
  • Backup and auditing

Role of DBA

  • Ensure security and backup
  • Optimize performance
  • Manage users and roles

Practical Topics

DDL and DML

  • DDL: CREATE, ALTER, DROP
  • DML: SELECT, INSERT, UPDATE, DELETE

SQL Data Types

  • CHAR, VARCHAR
  • BINARY, VARBINARY
  • TINYBLOB, TINYTEXT, TEXT, LONGTEXT
  • ENUM, BIT
  • TINYINT, BOOLEAN, INTEGER
  • FLOAT, DOUBLE, DECIMAL
  • DATE, DATETIME

Unit 1 â€“ Database Management System (DBMS)

1. What is Database and DBMS?

A Database is an organized collection of data. DBMS (Database Management System) is a software system that allows users to define, create, maintain, and control access to the database.

Advantages: Reduces data redundancy, provides data integrity, better security, easy access.

Disadvantages: Complex to set up, requires technical expertise, hardware costs.

2. Differentiate between file processing system and DBMS

  • DBMS offers better security and consistency than file systems.
  • DBMS supports multiple users; file system doesn't.
  • Data can be queried easily in DBMS using SQL.
  • Redundancy is reduced in DBMS.

3. Models of DBMS with advantages/disadvantages

  • Hierarchical Model: Tree-like structure. Fast access but rigid.
  • Network Model: Complex relationships but harder to manage.
  • Relational Model: Easy to use, flexible, most widely used.

4. What is a relational database?

It is a database that stores data in tables (relations). Each row is a record and each column is a field.

5. What is data redundancy?

It is the duplication of data. DBMS reduces redundancy by normalization and relational data storage.

6. Centralized vs Distributed database

  • Centralized: Data is stored at one location.
  • Distributed: Data is stored across multiple sites.

7. Who is a DBA?

DBA (Database Administrator) manages the database, handles backups, user access, tuning, and security.

8. Normalization

Normalization reduces redundancy and ensures data integrity.

  • 1NF: Atomic values only.
  • 2NF: No partial dependency.
  • 3NF: No transitive dependency.

9. Keys in DBMS

  • Primary key: Uniquely identifies records.
  • Foreign key: Links to primary key in another table.
  • Candidate key: Any field that can serve as primary key.

10. What is SQL?

SQL (Structured Query Language) is used to manage and query databases. Components include DDL, DML, DCL, TCL.

11. SQL Commands

-- a. Create table
CREATE TABLE students (
  Id INT,
  Name VARCHAR(50),
  Class VARCHAR(20),
  Marks INT
);

-- b. Insert records
INSERT INTO students VALUES (1, 'Rachita', '09A', 50);
INSERT INTO students VALUES (2, 'Riya', '12C', 15);

-- c. Display records
SELECT * FROM students;

-- d. Update marks
UPDATE students SET Marks = 95 WHERE Name = 'Rachita';

-- e. Delete record
DELETE FROM students WHERE Id = 2;

12. Define the following terms

  • Data Dictionary: Stores metadata.
  • Primary Key: Unique identifier.
  • Relationship: Link between tables.
  • DML: Data Manipulation Language - SELECT, INSERT, DELETE, UPDATE.
  • SQL: Language for managing databases.
  • Data Integrity: Accuracy and consistency of data.
  • DDL: Data Definition Language - CREATE, ALTER, DROP.
  • Data Security: Protection from unauthorized access.
  • Database System: Combination of DBMS and database.

Unit 2 â€“ C Programming

1. Working with a Function

  • Define Function: A reusable block of code.
  • Syntax:
    return_type function_name(parameters) { // body }
  • Types: Library functions (e.g., printf), User-defined functions.
  • Components:
    • Function prototype
    • Function call
    • Function definition
    • Return type

2. Categories of Functions with Example

  • With return, no argument:
    int getNumber() { return 5; }
  • With return, with arguments:
    int add(int a, int b) { return a + b; }
  • No return, no argument:
    void hello() { printf("Hi"); }
  • No return, with arguments:
    void printNum(int x) { printf("%d", x); }

3. Storage Classes

  • Automatic: Default local variables.
  • External: Global variables.
  • Register: Stored in CPU registers.
  • Static: Retains value across function calls.

4. Recursive Function

Example: Factorial

int factorial(int n) {
  if (n == 0) return 1;
  else return n * factorial(n - 1);
}

5. Passing Array to Function

void printArray(int arr[], int size) {
  for (int i = 0; i < size; i++) {
    printf("%d ", arr[i]);
  }
}
