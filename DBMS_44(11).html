Unit 1 – Database Management System (DBMS)

1. What is Database and DBMS?

A Database is a structured and organized collection of related data that is stored and retrieved efficiently. It is designed to store, manage, and provide access to large amounts of information.

A DBMS (Database Management System) is a comprehensive software suite that acts as an interface between the users/applications and the database. It allows users to define, create, query, update, maintain, and control access to the database.

Example: MySQL, PostgreSQL, Oracle, Microsoft SQL Server.

Advantages: Reduces data redundancy and inconsistency, provides better data integrity, enhanced security, supports complex queries, and enables efficient data sharing.

Disadvantages: Complex to set up and manage, requires specialized technical expertise (DBA), significant hardware and software costs, and potential performance overhead.

2. Differentiate between File Processing System and DBMS

  • Data Redundancy and Inconsistency: File systems often have high redundancy and inconsistency as data is duplicated across different files. DBMS minimizes redundancy through centralized control and normalization.
  • Data Integrity: DBMS provides mechanisms (like constraints) to ensure data accuracy and consistency, which is difficult to enforce in file systems.
  • Concurrency Control: DBMS supports multiple users accessing and updating data simultaneously without conflicts (concurrency control). File systems lack robust concurrency control.
  • Security: DBMS offers granular security features (e.g., user roles, permissions) for data protection, unlike simple file systems.
  • Data Access and Querying: Data in DBMS can be easily queried and manipulated using standard languages like SQL. Accessing data in file systems often requires writing specific application programs for each task.

3. Models of DBMS with advantages/disadvantages

  • Hierarchical Model: Data is organized in a tree-like structure (parent-child relationship).
    • Advantage: Fast access for specific queries along the hierarchy.
    • Disadvantage: Rigid structure, difficult to represent complex relationships, and adding new relationships is challenging.
  • Network Model: An extension of the hierarchical model, allowing a record to have multiple parent and child relationships (many-to-many).
    • Advantage: More flexible than the hierarchical model; handles complex relationships efficiently.
    • Disadvantage: Structurally complex and difficult to manage and implement.
  • Relational Model: Data is stored in two-dimensional tables (relations), with rows representing records (tuples) and columns representing attributes (fields).
    • Advantage: Simple, easy to use, highly flexible, and supported by powerful query languages (SQL).
    • Disadvantage: Can be less efficient for handling extremely large datasets compared to specialized models.

4. What is a Relational Database?

A relational database is a type of database based on the relational model, where data is organized into tables (relations). These tables are connected through common fields, allowing data to be linked logically. The primary focus of a relational database is data integrity and consistency.

Example: A database for a school might have a 'Students' table and a 'Courses' table, linked by a student ID.

5. What is Data Redundancy?

Data redundancy is the duplication of data within a database or system. It occurs when the same information is stored multiple times in different locations. While sometimes intentional for backups, it generally leads to inconsistencies and wasted storage.

DBMS reduces redundancy primarily through Normalization, a process of organizing data to eliminate duplication and improve data integrity.

6. Centralized vs Distributed database

  • Centralized Database: All data is stored, maintained, and processed at a single location or on a single computer system.
    • Advantage: Easy to manage, control, and secure.
    • Disadvantage: Single point of failure; performance bottlenecks for geographically dispersed users.
  • Distributed Database: Data is stored across multiple physically dispersed sites or nodes, connected by a network. Users perceive it as a single logical database.
    • Advantage: Improved availability, faster response times, and resilience against node failures.
    • Disadvantage: Increased complexity in managing synchronization, consistency, and security.

7. Who is a DBA?

A DBA (Database Administrator) is an IT professional responsible for the maintenance, security, performance, and operational integrity of a database. Key responsibilities include installing and configuring the DBMS, managing user access and permissions, performing backups and recovery, and optimizing database performance (tuning).

8. Normalization

Normalization is a systematic process of organizing data in a database to reduce data redundancy and improve data integrity. It aims to eliminate anomalous behavior during data updates, insertions, and deletions.

  • 1NF (First Normal Form): Ensures that all attributes contain only atomic (indivisible) values and that there are no repeating groups.
  • 2NF (Second Normal Form): Requires the relation to be in 1NF and all non-key attributes must be fully functionally dependent on the primary key (applies to relations with composite keys).
  • 3NF (Third Normal Form): Requires the relation to be in 2NF and eliminates transitive dependencies (where a non-key attribute depends on another non-key attribute).

9. Keys in DBMS

Keys are essential fields or sets of fields used to establish relationships between tables and uniquely identify rows within a table.

  • Primary Key: A column or set of columns that uniquely identifies each record in a table. It cannot contain NULL values and must be unique. (e.g., Student ID in a Students table).
  • Foreign Key: A column or set of columns in one table that references the primary key of another table. It establishes a link between the two tables. (e.g., Department ID in an Employees table referencing the primary key of a Departments table).
  • Candidate Key: Any attribute or set of attributes that can uniquely identify a tuple (row) in a relation. All candidate keys are potential primary keys.
  • Super Key: Any set of attributes that uniquely identifies a tuple. A primary key is a minimal super key.

10. What is SQL?

SQL (Structured Query Language) is a standardized programming language designed for managing relational databases and performing various operations on the data within them. SQL is used to create, retrieve, update, and delete database records.

Components:

  • DDL (Data Definition Language): Used for defining the database schema (e.g., CREATE, ALTER, DROP).
  • DML (Data Manipulation Language): Used for managing data within the schema (e.g., SELECT, INSERT, UPDATE, DELETE).
  • DCL (Data Control Language): Used for controlling access to data (e.g., GRANT, REVOKE).
  • TCL (Transaction Control Language): Used for managing transactions (e.g., COMMIT, ROLLBACK).

12. Define the following terms

  • Data Dictionary: A centralized repository of information about data, such as schema, constraints, data types, and access controls. It stores metadata (data about data).
  • Primary Key: A unique identifier for records in a table (defined in Q9).
  • Relationship: A logical association or connection between two or more tables in a relational database, typically established using foreign keys.
  • DML (Data Manipulation Language): Commands used for retrieving, inserting, deleting, and updating data within a database (e.g., SELECT, INSERT, UPDATE, DELETE).
  • SQL (Structured Query Language): A standardized language for managing and querying relational databases (defined in Q10).
  • Data Integrity: The accuracy, consistency, and reliability of data over its entire lifecycle. Constraints and rules are used to maintain data integrity.
  • DDL (Data Definition Language): Commands used to define or modify the database structure or schema (e.g., CREATE, ALTER, DROP).
  • Data Security: Measures taken to protect the database from unauthorized access, modification, or destruction. This includes authentication, authorization, and encryption.
  • Database System: An integrated system comprising the database (data itself), the DBMS software, and the related applications and users.

Unit 2 – C Programming

1. Working with a Function

A Function in C is a self-contained block of code designed to perform a specific task. Functions promote code reusability, modularity, and easier debugging.

  • Definition: The actual body of the function that performs the task.
  • Syntax: C Program: Square of a Number

    C Program to Find Square of a Number

    Executed by: Radha Jha

    #include <stdio.h>

// Function declaration
int square(int num);

int main() {
    int number, result;

    printf("Enter a number: ");
    scanf("%d", &number);

    result = square(number); // Function call

    printf("Square of %d is %d\n", number, result);
    printf("Executed by Radha Jha");
    return 0;
}

// Function definition
int square(int num) 
{
    return num * num;
}
  • Types:
    • Library Functions: Predefined functions provided by the C standard library (e.g., printf(), scanf(), sqrt()).
    • User-defined Functions: Functions created by the programmer to perform specific tasks.
  • Components:
    • Function Prototype (Declaration): Declares the function's return type, name, and parameters to the compiler before it is defined.
    • Function Call: Invokes the function to execute its code.
    • Function Definition: Contains the actual code for the function's logic.
    • Return Type: Specifies the type of value the function returns after execution (void if it returns nothing).

2. Categories of Functions with Example

  • 1. No Arguments and No Return Value (void function): 
    void hello() {
  printf("Hello, World!");
}
  • 2. No Arguments but Returns a Value: 
    int getNumber() {
  return 42;
}
  • 3. With Arguments but No Return Value: 
    void printNum(int x) {
  printf("The number is: %d", x);
}
  • 4. With Arguments and Returns a Value: 
    int add(int a, int b) {
  return a + b;
}

3. Storage Classes

Storage classes in C determine the scope, visibility, and lifetime of variables and functions.

  • Automatic (auto): The default storage class for local variables. Variables are created when the function is called and destroyed when the function exits. They are stored in the stack.
  • External (extern): Used to declare a global variable defined in another file, making it accessible across multiple source files.
  • Register (register): Suggests to the compiler that the variable should be stored in a CPU register for faster access, although the compiler may ignore this request. Used for frequently accessed variables.
  • Static (static):
    • Local static variables: Retain their values between function calls. They are initialized only once.
    • Global static variables/functions: Their scope is limited to the file in which they are defined.
C Code: Factorial using Recursion

C Program: Factorial Calculation using Recursion

#include <stdio.h>

// Function prototype for the recursive factorial calculation
long long factorial(int n);

int main() {
    int num;
    long long result;

    printf("Enter a non-negative integer: ");
    scanf("%d", &num);

    // Check if the number is negative
    if (num < 0) {
        printf("Factorial is not defined for negative numbers.\n");
    } else {
        result = factorial(num);
        printf("Factorial of %d = %lld\n", num, result);
    }
    printf("EXECUTED BY RADHA JHA");
    return 0;
}
// Recursive function to calculate the factorial
long long factorial(int n) {
    // Base case: Factorial of 0 and 1 is 1
    if (n == 0 || n == 1) {
        return 1;
    } 
    // Recursive step: n * factorial(n-1)
    else {
        return n * factorial(n - 1);
    }
}

5. Passing Array to Function

In C, when an array is passed to a function, it is typically passed by reference (the address of the array's first element is passed), rather than by value. This allows the function to modify the original array.

Example: Printing an Array

void printArray(int arr[], int size) {
  printf("Array elements: ");
  for (int i = 0; i < size; i++) {
    printf("%d ", arr[i]);
  }
  printf("\n");
}

int main() {
  int myArray[] = {10, 20, 30, 40, 50};
  int size = sizeof(myArray) / sizeof(myArray[0]);
  printArray(myArray, size);
  return 0;
}
